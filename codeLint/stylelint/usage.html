# 安装与使用

# 基础用法

  1. 安装 Stylelint 及其标准配置
npm install --save-dev stylelint stylelint-config-standard
  1. 在项目根目录创建.stylelintrc.json配置文件
{
  "extends": "stylelint-config-standard"
}
  1. 在项目中对所有 CSS 文件执行 Stylelint
npx stylelint "**/*.css"

提示

如果你在使用 Stylelint 的同时使用 Prettier，你需要关闭所有冲突的规则。推荐使用 Prettier 共享配置来实现这个效果。

npm install --save-dev stylelint-config-prettier

{
  "extends": ["stylelint-config-standard", "stylelint-config-prettier"]
}

# 约束其他样式语言

你需要使用社区编写的自定义语法。

# 使用社区共享配置

我们建议扩展一个共享配置，其中包含适合您首选语言或库的适当语法。例如，您可以扩展stylelint-config-standard-scss (opens new window)共享配置来检测SCSS。

  1. 使用 npm 安装 Stylelint 和标准配置
npm install --save-dev stylelint stylelint-config-standard-scss
  1. 在项目根目录创建.stylelintrc.json配置文件
{
  "extends": "stylelint-config-standard-scss"
}

该配置包括postcss-scss语法，配置 SCSS 的内置规则，并包括stylelint-scss插件(一组特定于 SCSS 的规则)。

如何结合 Prettier 使用，你应该使用他们对于 SCSS 的共享配置 (opens new window)

{
  "extends": ["stylelint-config-standard-scss", "stylelint-config-prettier-scss"]
}

其他共享配置包括

  • stylelint-config-html (opens new window)
  • stylelint-config-recommended-vue (opens new window)

# 直接使用自定义语法

如果共享配置对于首选语言或库不可用，那么您可以自己安装适当的自定义语法，并使用customSyntax选项来配置它。

例如，分析Lit Elements (opens new window)中的 CSS:

  1. 使用 npm 安装 Stylelint 及其标准配置和postcss-lit (opens new window)
npm install --save-dev stylelint stylelint-config-standard postcss-lit
  1. 在项目根目录创建.stylelintrc.json配置文件
{
  "extends": "stylelint-config-standard",
  "customSyntax": "postcss-lit"
}

其他与 Stylelint 兼容的 PostCSS 语法包括

  • postcss-markdown (opens new window)
  • postcss-less (opens new window)
  • postcss-sass (opens new window)
  • sugarss (opens new window)

更多请查看PostCSS (opens new window)

# 使用多个自定义语法

您可以使用override属性。例如，要检测 CSS 文件和 Lit Elements 中的 CSS，你可以更新你的配置对象，包括:

{
  "extends": ["stylelint-config-standard"],
  "overrides": [
    {
      "files": ["**/*.{js}"],
      "customSyntax": "postcss-lit"
    }
  ]
}
