# 配置项列表

标记说明

💪 高级功能；💥 危险地带；⭐️ 实验性功能

  • external: 配置外部依赖项
  • input: 配置构建的入口点
  • output: 配置输出
    • dir: 存放所有生成的chunk的目录(单个chunk可使用file取代)
    • file: 要写入的文件
    • format: 指定生成的包的格式
    • globals: 配置全局变量
    • name: 配置包的全局变量名(对iife/umd必备)
    • plugins: 配置特定于输出的插件
    • 💪 assetFileNames: 对 assets 资源自定义命名
    • 💪 chunkFileNames: 对代码拆分时创建的共享块自定义命名
    • 💪 entryFileNames: 对入口文件自定义命名
    • 💪 banner/footer: 在 bundle 前面/后面追加字符串
    • 💪 compact: 是否压缩由 rollup 生成的包装器代码
    • 💪 extend: 是否扩展在umd或iife格式下由name属性定义的全局变量
    • 💪 generatedCode: Rollup 可以在生成的代码中安全地使用哪些语言特性
      • preset: 允许对预设覆盖一些选项
      • arrowFunctions: 是否为自动生成的代码段使用箭头函数
      • constBindings: 在某些地方和帮助函数中使用const而不是var
      • objectShorthand: 当属性名与值匹配时，允许在对象中使用简写符号
      • reservedNamesAsProps: 保留字是否可以在不使用引号的情况下用作属性名
      • symbols: 是否允许在自动生成的代码片段中使用Symbol
    • 💪 hoistTransitiveImports: 是否入口块的递归导入可以作为空导入被添加到其他入口块中
    • 💪 inlineDynamicImports: 使用内联动态导入
    • 💪 interop: 控制 Rollup 处理 default、namespace 及其动态导入的方式
    • 💪 intro/outro: 在特定于格式的包装器中添加代码
    • 💪 manualChunks: 创建自定义共享公共块
    • 💪 minifyInternalExports: 将内部变量导出为单个字母变量
    • 💪 paths: 将外部模块 id 映射到路径
    • 💪 preserveModules (opens new window): 使用原始模块名作为文件名，为所有模块创建单独的块
    • 💪 preserveModulesRoot (opens new window): 一个应该从output.dir路径中删除的入口模块的目录路径
    • 💪 sourcemap (opens new window): sourcemap 的创建模式
    • 💪 sourcemapExcludeSources (opens new window): 禁止源代码被添加到 sourcemap 中
    • 💪 sourcemapFile (opens new window): 生成 sourcemap 的位置
    • 💪 sourcemapPathTransform (opens new window): 应用于 sourcemap 中每个路径的转换
    • 💪 validate (opens new window): 检测生成的代码是否是有效的 JavaScript
    • 💥 amd (opens new window)
      • id: 用于 AMD/UMD 捆绑的 ID
      • autoId: 设置 ID 为 chunk ID
      • basePath: 作为自动生成的 ID 的前缀路径
      • define: 取代define的函数名
    • 💥 esModule (opens new window): 当生成非 ES 格式的导出时，是否添加__esModule: true属性
    • 💥 exports (opens new window): 导出模式
    • 💥 externalLiveBindings (opens new window): 生成支持外部导入的实时绑定的代码
    • 💥 freeze (opens new window): 是否使用Object.freeze()命名空间导入动态访问的对象
    • 💥 indent (opens new window): 要使用的缩进字符串
    • 💥 noConflict (opens new window): 这将生成一个额外的noConflict输出到 UMD 包
    • 💥 preferConst (opens new window): 为导出生成const声明，而不是为var声明
    • 💥 sanitizeFileName (opens new window): 消除 chunk 名字中的特殊字符(\0, ?, *)
    • 💥 strict (opens new window): 是否在生成的非 es 包的顶部包含'use strict'
    • 💥 systemNullSetters (opens new window): 当输出system模块格式时，这将用null替换空的 setter 函数
  • plugins: 配置插件
  • 💪 cache: 缓存
  • 💪 makeAbsoluteExternalsRelative: 将输出中绝对外部路径转换为相对路径
  • 💪 maxParallelFileReads: 限制汇总读取模块时并行打开的文件数量
  • 💪 onwarn: 拦截警告消息的函数
  • 💪 preserveEntrySignatures (opens new window): 控制 Rollup 确保入口文件是否应该像底层入口模块一样导出的方式
  • 💪 strictDeprecations (opens new window): 当使用弃用的特性时，Rollup 将抛出错误而不是显示警告
  • 💥 acorn (opens new window): acornjs 配置，详情参阅Acorn 文档 (opens new window)
  • 💥 acornInjectPlugins (opens new window): acornjs 插件
  • 💥 context (opens new window): 模块的上下文
  • 💥 moduleContext (opens new window): 按 id 区分不同模块的上下文
  • 💥 preserveSymlinks (opens new window): 解析文件时遵循符号链接
  • 💥 shimMissingExports (opens new window): 从没有定义绑定的文件导入绑定是否失败
  • 💥 treeshake (opens new window): 是否应用 tree-shaking 和微调 tree-shaking 过程
    • 💥 preset: 选择准备覆盖部分选项的预设
    • 💥 annotations: 是否忽略来自纯注释的提示
    • 💥 correctVarValueBeforeDeclaration: Rollup 是否对 var 声明的变量的值做常量假设
    • 💥 moduleSideEffects: 禁止空导入
    • 💥 propertyReadSideEffects: 是否将读取但为使用的对象属性作为副作用(side effects)
    • 💥 tryCatchDeoptimization: 是否支持依赖于那些被抛出的错误的特征检测工作流
    • 💥 unknownGlobalSideEffects: Rollup 是否保留对非内置全局变量的任何访问
  • ⭐️ experimentalCacheExpiry (opens new window): 确定在运行了多少次缓存后插件不再使用的资产应该被删除
  • ⭐️ perf (opens new window): 是否收集performance timings
  • watch (opens new window): 监视模式
    • buildDelay (opens new window): 配置 Rollup 在触发重新生成之前等待进一步更改的时间(以毫秒为单位)
    • chokidar (opens new window): chokidar (opens new window)选项
    • clearScreen (opens new window): 触发重构时是否清除屏幕
    • exclude (opens new window): 防止文件被监视
    • include (opens new window): 将文件监视限制在某些文件
    • skipWrite (opens new window): 是否在触发重新构建时跳过bundle.write()步骤
